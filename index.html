1. Datos iniciales

Nombre del estudiante: Juan García

Título del proyecto: Página Web sobre Analog Horror (Informe Técnico)

Propósito: Explorar el fenómeno del "Analog Horror" (terror de baja fidelidad), destacando su impacto cultural y analizando sus elementos narrativos clave. Es una demostración de la aplicación de estándares web.

2. Estructura y Contenido

La estructura del sitio es semántica, utilizando etiquetas clave de HTML5 para una lectura óptima por máquinas y lectores de pantalla.

  • Etiquetas Semánticas: Uso de <header>, <nav>, <main>, <section>, y <footer>.
  • Contenido: Se incluye texto relevante sobre el origen del género, y se utilizan listas (<ul>) y párrafos (<p>) para organizar la información.

Ejemplo de Contenido (Texto e Imágenes)

El Analog Horror aprovecha la nostalgia y el miedo a la pérdida de control en los medios antiguos, usando fallos en la imagen (glitches) y mensajes ambiguos para generar pánico.

Enlace externo de referencia: Análisis del Analog Horror

3. Diseño y Estilo (CSS)

El diseño se basó en los principios de estética retro-digital (CRT y VHS) y la necesidad de accesibilidad (contraste). El código CSS se encuentra en el archivo style.css.

  • Paleta de Colores: Uso de #1a1a1a (Fondo Oscuro), #f4f4f4 (Fondos de Bloque), y #8b0000 (Rojo oscuro para acentos de Terror).
  • Tipografía: 'VT323' (estilo retro/terminal) para títulos y 'Arial' o 'Verdana' para el cuerpo (Requisito: Tipografía legible).

4. Usabilidad y Accesibilidad

Se priorizó la experiencia de usuario y el cumplimiento de estándares de accesibilidad.

  • Contraste Adecuado: Se garantiza el contraste para lectura (Requisito: Contraste).
  • Texto Alternativo: Todas las etiquetas <img> incluyen un atributo alt descriptivo (Requisito: Texto alternativo).
  • Tamaño de Fuente: Se usa `rem` o porcentajes en CSS para que el texto sea escalable (Requisito: Tamaños de fuente).

5. Aspectos Técnicos

El proyecto se construyó bajo las mejores prácticas de ingeniería de software.

  • Tecnologías Usadas: HTML5, CSS3 y JavaScript (Funcionalidad).
  • Separación de Capas: Uso de archivos style.css y script.js externos para una mayor mantenibilidad.

Fragmento de Funcionalidad JavaScript

Se implementó un script que añade un efecto de "glitch" visual a un elemento al pasar el ratón, mejorando la inmersión en el tema Analog Horror.


// Ejemplo de JavaScript en script.js
document.getElementById('glitch-box').addEventListener('mouseover', function() {
    this.classList.add('active-glitch');
});

6. Proceso de Construcción

  1. Conceptualización: Definición de la temática y paleta de colores.
  2. Estructuración: Creación del HTML semántico.
  3. Diseño Visual: Implementación del CSS para diseño de la interfaz y responsividad.
  4. Pruebas y Funcionalidad: Implementación de JavaScript y verificación de la accesibilidad.

Dificultades y Resoluciones

Dificultad: Asegurar que la estética de baja fidelidad no comprometiera el **Contraste** y la **Accesibilidad** del texto.

Resolución: Se usaron bloques de contenido en color claro sobre el fondo oscuro, manteniendo un contraste óptimo.

7. Conclusión

Aprendizajes Clave: Se reforzó la comprensión de que la estructura semántica (HTML) es la base de la accesibilidad, y que la modularidad (CSS/JS externo) es vital para la mantenibilidad.

Mejoras Futuras

Se planea integrar WebGL para crear efectos visuales de distorsión más realistas.
